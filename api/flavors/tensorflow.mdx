title: mlflow.tensorflow
description: API reference for MLflow's TensorFlow flavor
---

The `mlflow.tensorflow` module provides an API for logging and loading TensorFlow models. This module exports TensorFlow models with the following flavors:

- **TensorFlow (native) format** - The main flavor that can be loaded back into TensorFlow
- **mlflow.pyfunc** - Produced for use by generic pyfunc-based deployment tools and batch inference

<Note>
If you log a Keras or TensorFlow model without a signature, inference with `mlflow.pyfunc.spark_udf()` will not work unless the model's pyfunc representation accepts pandas DataFrames as inference inputs.
</Note>

## Saving Models

### save_model()

Save a TF2 core model (inheriting tf.Module) or Keras model to a path on the local file system.

```python
mlflow.tensorflow.save_model(
    model,
    path,
    conda_env=None,
    code_paths=None,
    mlflow_model=None,
    custom_objects=None,
    signature=None,
    input_example=None,
    pip_requirements=None,
    extra_pip_requirements=None,
    saved_model_kwargs=None,
    keras_model_kwargs=None,
    metadata=None,
    extra_files=None
)
```

<ParamField path="model" type="tf.Module | keras.Model" required>
  The TensorFlow module or Keras model to be saved.
</ParamField>

<ParamField path="path" type="str" required>
  Local path where the MLflow model is to be saved.
</ParamField>

<ParamField path="custom_objects" type="dict">
  A Keras `custom_objects` dictionary mapping names (strings) to custom classes or functions. MLflow saves these using CloudPickle and restores them automatically.
</ParamField>

<ParamField path="signature" type="ModelSignature">
  Model signature describing model inputs and outputs. All fields must be TensorSpec with first dimension as -1 (variable dimension).
</ParamField>

<ParamField path="saved_model_kwargs" type="dict">
  Dict of kwargs to pass to `tensorflow.saved_model.save` method (for tf.Module models).
</ParamField>

<ParamField path="keras_model_kwargs" type="dict">
  Dict of kwargs to pass to `keras_model.save` method (for Keras models).
</ParamField>

### log_model()

Log a TensorFlow or Keras model in MLflow Model format.

```python
mlflow.tensorflow.log_model(
    model,
    artifact_path=None,
    custom_objects=None,
    conda_env=None,
    code_paths=None,
    signature=None,
    input_example=None,
    registered_model_name=None,
    await_registration_for=300,
    pip_requirements=None,
    extra_pip_requirements=None,
    saved_model_kwargs=None,
    keras_model_kwargs=None,
    metadata=None,
    extra_files=None,
    name=None,
    params=None,
    tags=None,
    model_type=None,
    step=0,
    model_id=None
)
```

<ParamField path="model" type="tf.Module | keras.Model" required>
  The TensorFlow or Keras model to be logged.
</ParamField>

<ParamField path="name" type="str">
  Name of the logged model.
</ParamField>

<ParamField path="registered_model_name" type="str">
  If given, create a model version under this registered model name.
</ParamField>

**Returns:** `ModelInfo` instance containing metadata of the logged model.

## Loading Models

### load_model()

Load an MLflow model that contains the TensorFlow flavor from the specified path.

```python
mlflow.tensorflow.load_model(
    model_uri,
    dst_path=None,
    saved_model_kwargs=None,
    keras_model_kwargs=None
)
```

<ParamField path="model_uri" type="str" required>
  The location, in URI format, of the MLflow model. Supported URI schemes:
  - `/Users/me/path/to/local/model`
  - `s3://my_bucket/path/to/model`
  - `runs:/<mlflow_run_id>/run-relative/path/to/model`
  - `models:/<model_name>/<model_version>`
</ParamField>

<ParamField path="saved_model_kwargs" type="dict">
  kwargs to pass to `tensorflow.saved_model.load` method (for TensorFlow 2 core models).
</ParamField>

<ParamField path="keras_model_kwargs" type="dict">
  kwargs to pass to `keras.models.load_model` method (for Keras models).
</ParamField>

**Returns:** A callable graph (tf.function) that takes inputs and returns inferences.

## Autologging

### autolog()

Enables autologging for `tf.keras`.

```python
mlflow.tensorflow.autolog(
    log_models=True,
    log_datasets=True,
    disable=False,
    exclusive=False,
    disable_for_unsupported_versions=False,
    silent=False,
    registered_model_name=None,
    log_input_examples=False,
    log_model_signatures=True,
    saved_model_kwargs=None,
    keras_model_kwargs=None,
    extra_tags=None,
    log_every_epoch=True,
    log_every_n_steps=None,
    checkpoint=True,
    checkpoint_monitor="val_loss",
    checkpoint_mode="min",
    checkpoint_save_best_only=True,
    checkpoint_save_weights_only=False,
    checkpoint_save_freq="epoch"
)
```

<Note>
Only TensorFlow >= 2.3 is supported.
</Note>

<ParamField path="log_every_epoch" type="bool" default="True">
  If True, training metrics are logged at the end of each epoch.
</ParamField>

<ParamField path="checkpoint" type="bool" default="True">
  Enable automatic model checkpointing.
</ParamField>

### Logged Information

**Metrics and Parameters:**
- Training and validation loss
- User-specified metrics
- Optimizer configuration (learning_rate, momentum, etc.)
- Training configurations (epochs, batch_size, etc.)

**Artifacts:**
- Model summary on training start
- Saved Keras model in MLflow Model format
- TensorBoard logs on training end

**EarlyStopping:**
- Metrics from `EarlyStopping` callbacks
- Parameters associated with `EarlyStopping`

## Example - Creating Model Signature

```python
from mlflow.types.schema import Schema, TensorSpec
from mlflow.models import ModelSignature
import numpy as np

input_schema = Schema([
    TensorSpec(np.dtype(np.uint64), (-1, 5), "field1"),
    TensorSpec(np.dtype(np.float32), (-1, 3, 2), "field2"),
])

# Create signature for model requiring 2 inputs
signature = ModelSignature(inputs=input_schema)

# Log model with signature
with mlflow.start_run():
    mlflow.tensorflow.log_model(
        model,
        name="tf_model",
        signature=signature
    )
```
