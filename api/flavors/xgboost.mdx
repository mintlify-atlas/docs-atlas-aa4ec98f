title: mlflow.xgboost
description: API reference for MLflow's XGBoost flavor

The `mlflow.xgboost` module provides an API for logging and loading XGBoost models. This module exports XGBoost models with the following flavors:

- **XGBoost (native) format** - The main flavor that can be loaded back into XGBoost
- **mlflow.pyfunc** - Produced for use by generic pyfunc-based deployment tools and batch inference

## Saving Models

### save_model()

Save an XGBoost model to a path on the local file system.

```python
mlflow.xgboost.save_model(
    xgb_model,
    path,
    conda_env=None,
    code_paths=None,
    mlflow_model=None,
    signature=None,
    input_example=None,
    pip_requirements=None,
    extra_pip_requirements=None,
    model_format="ubj",
    metadata=None,
    extra_files=None,
    **kwargs
)
```

<ParamField path="xgb_model" type="xgboost.Booster | XGBModel" required>
  XGBoost model to be saved. Can be either `xgboost.Booster` or models that implement the scikit-learn API.
</ParamField>

<ParamField path="path" type="str" required>
  Local path where the model is to be saved.
</ParamField>

<ParamField path="model_format" type="str" default="ubj">
  File format for saving the model. Options:
  - `"ubj"` - UBJSON format (recommended for performance and cross-platform compatibility)
  - `"json"` - JSON format
  - `"xgb"` - Native XGBoost binary format
</ParamField>

<ParamField path="signature" type="ModelSignature">
  Model signature describing model inputs and outputs.
</ParamField>

<ParamField path="input_example" type="pd.DataFrame | np.ndarray">
  Example input for model testing and signature inference.
</ParamField>

### log_model()

Log an XGBoost model as an MLflow artifact for the current run.

```python
mlflow.xgboost.log_model(
    xgb_model,
    artifact_path=None,
    conda_env=None,
    code_paths=None,
    registered_model_name=None,
    signature=None,
    input_example=None,
    await_registration_for=300,
    pip_requirements=None,
    extra_pip_requirements=None,
    model_format="ubj",
    metadata=None,
    extra_files=None,
    name=None,
    params=None,
    tags=None,
    model_type=None,
    step=0,
    model_id=None,
    **kwargs
)
```

<ParamField path="xgb_model" type="xgboost.Booster | XGBModel" required>
  XGBoost model to be logged.
</ParamField>

<ParamField path="name" type="str">
  Name of the logged model.
</ParamField>

<ParamField path="registered_model_name" type="str">
  If given, create a model version under this registered model name.
</ParamField>

**Returns:** `ModelInfo` instance containing metadata of the logged model.

## Loading Models

### load_model()

Load an XGBoost model from a local file or a run.

```python
mlflow.xgboost.load_model(
    model_uri,
    dst_path=None
)
```

<ParamField path="model_uri" type="str" required>
  The location, in URI format, of the MLflow model. Supported URI schemes:
  - `/Users/me/path/to/local/model`
  - `s3://my_bucket/path/to/model`
  - `runs:/<mlflow_run_id>/run-relative/path/to/model`
  - `models:/<model_name>/<model_version>`
</ParamField>

**Returns:** An XGBoost model (instance of `xgboost.Booster` or XGBoost scikit-learn model).

## Autologging

### autolog()

Enables (or disables) and configures autologging from XGBoost to MLflow.

```python
mlflow.xgboost.autolog(
    importance_types=None,
    log_input_examples=False,
    log_model_signatures=True,
    log_models=True,
    log_datasets=True,
    disable=False,
    exclusive=False,
    disable_for_unsupported_versions=False,
    silent=False,
    registered_model_name=None,
    model_format="ubj",
    extra_tags=None
)
```

<ParamField path="importance_types" type="list[str]">
  Importance types to log. If unspecified, defaults to `["weight"]`.
</ParamField>

<ParamField path="log_models" type="bool" default="True">
  If True, trained models are logged as MLflow model artifacts.
</ParamField>

<ParamField path="log_datasets" type="bool" default="True">
  If True, train and validation dataset information is logged to MLflow Tracking.
</ParamField>

<Note>
The scikit-learn API is now supported for autologging.
</Note>

### Logged Information

Autologging captures:
- Parameters specified in `xgboost.train`
- Metrics on each iteration (if `evals` specified)
- Metrics at the best iteration (if `early_stopping_rounds` specified)
- Feature importance as JSON files and plots
- Trained model with input example and inferred signature

## Example

```python
import xgboost as xgb
import mlflow
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split

# Enable autologging
mlflow.xgboost.autolog()

# Load data
iris = load_iris()
X_train, X_test, y_train, y_test = train_test_split(
    iris.data, iris.target, test_size=0.2
)

# Create DMatrix
dtrain = xgb.DMatrix(X_train, label=y_train)
dtest = xgb.DMatrix(X_test, label=y_test)

# Train model
with mlflow.start_run():
    params = {
        'objective': 'multi:softmax',
        'num_class': 3,
        'max_depth': 3
    }
    model = xgb.train(
        params,
        dtrain,
        num_boost_round=10,
        evals=[(dtest, 'test')]
    )
```
