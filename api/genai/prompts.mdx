The MLflow Prompts API provides a comprehensive framework for creating, managing, and version-controlling prompts with MLflow's robust model tracking framework.

## Prompt Management

### mlflow.genai.register_prompt

Register a new prompt in the MLflow Prompt Registry.

```python
def register_prompt(
    name: str,
    template: str | list[dict[str, Any]],
    commit_message: str | None = None,
    tags: dict[str, str] | None = None,
    response_format: type[BaseModel] | dict[str, Any] | None = None,
    model_config: PromptModelConfig | dict[str, Any] | None = None,
) -> PromptVersion
```

<ParamField path="name" type="str" required>
  The name of the prompt.
</ParamField>

<ParamField path="template" type="str | list[dict[str, Any]]" required>
  The template content of the prompt. Can be either:
  
  - **A string** containing text with variables enclosed in double curly braces, e.g. `{{variable}}`, which will be replaced with actual values by the `format` method.
  - **A list of dictionaries** representing chat messages, where each message has 'role' and 'content' keys (e.g., `[{"role": "user", "content": "Hello {{name}}"}]`)
  
  <Note>
  If you want to use the prompt with a framework that uses single curly braces (e.g., LangChain), you can use the `to_single_brace_format()` method to convert the loaded prompt.
  </Note>
</ParamField>

<ParamField path="commit_message" type="str" optional>
  A message describing the changes made to the prompt, similar to a Git commit message.
</ParamField>

<ParamField path="tags" type="dict[str, str]" optional>
  A dictionary of tags associated with the **prompt version**. This is useful for storing version-specific information, such as the author of the changes.
</ParamField>

<ParamField path="response_format" type="type[BaseModel] | dict[str, Any]" optional>
  Optional Pydantic class or dictionary defining the expected response structure. This can be used to specify the schema for structured outputs from LLM calls.
</ParamField>

<ParamField path="model_config" type="PromptModelConfig | dict[str, Any]" optional>
  Optional PromptModelConfig instance or dictionary containing model-specific configuration including model name and settings like temperature, top_p, max_tokens. Using PromptModelConfig provides validation and type safety for common parameters.
  
  Example (dict): `{"model_name": "gpt-4", "temperature": 0.7}`
  
  Example (PromptModelConfig): `PromptModelConfig(model_name="gpt-4", temperature=0.7)`
</ParamField>

<ResponseField name="return" type="PromptVersion">
  A `PromptVersion` object that was created.
</ResponseField>

#### Example

```python
import mlflow

# Register a text prompt
mlflow.genai.register_prompt(
    name="greeting_prompt",
    template="Respond to the user's message as a {{style}} AI.",
)

# Register a chat prompt with multiple messages
mlflow.genai.register_prompt(
    name="assistant_prompt",
    template=[
        {"role": "system", "content": "You are a helpful {{style}} assistant."},
        {"role": "user", "content": "{{question}}"},
    ],
    response_format={"type": "object", "properties": {"answer": {"type": "string"}}},
)

# Load and use the prompt
prompt = mlflow.genai.load_prompt("greeting_prompt")

# Use the prompt in your application
import openai

openai_client = openai.OpenAI()
openai_client.chat.completion.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "system", "content": prompt.format(style="friendly")},
        {"role": "user", "content": "Hello, how are you?"},
    ],
)

# Update the prompt with a new version
prompt = mlflow.genai.register_prompt(
    name="greeting_prompt",
    template="Respond to the user's message as a {{style}} AI. {{greeting}}",
    commit_message="Add a greeting to the prompt.",
    tags={"author": "Bob"},
)
```

<Note>
If there is no registered prompt with the given name, a new prompt will be created. Otherwise, a new version of the existing prompt will be created.
</Note>

### mlflow.genai.load_prompt

Load a prompt from the MLflow Prompt Registry.

```python
def load_prompt(
    name_or_uri: str,
    version: str | int | None = None,
    allow_missing: bool = False,
    link_to_model: bool = True,
    model_id: str | None = None,
    cache_ttl_seconds: float | None = None,
) -> PromptVersion
```

<ParamField path="name_or_uri" type="str" required>
  The name of the prompt, or the URI in the format `"prompts:/name/version"`.
</ParamField>

<ParamField path="version" type="str | int" optional>
  The version of the prompt (required when using name, not allowed when using URI).
</ParamField>

<ParamField path="allow_missing" type="bool" default={false}>
  If True, return None instead of raising Exception if the specified prompt is not found.
</ParamField>

<ParamField path="link_to_model" type="bool" default={true}>
  If True, link the prompt to the model.
</ParamField>

<ParamField path="model_id" type="str" optional>
  The ID of the model to link the prompt to. Only used if `link_to_model` is True.
</ParamField>

<ParamField path="cache_ttl_seconds" type="float" optional>
  Time-to-live in seconds for the cached prompt. If not specified, uses the value from:
  - `MLFLOW_ALIAS_PROMPT_CACHE_TTL_SECONDS` environment variable for alias-based prompts (default 60)
  - `MLFLOW_VERSION_PROMPT_CACHE_TTL_SECONDS` environment variable for version-based prompts (default None, no TTL)
  
  Set to 0 to bypass the cache and always fetch from the server.
</ParamField>

<ResponseField name="return" type="PromptVersion">
  A `PromptVersion` object.
</ResponseField>

#### Example

```python
import mlflow

# Load the latest version of the prompt
prompt = mlflow.genai.load_prompt("my_prompt")

# Load a specific version of the prompt
prompt = mlflow.genai.load_prompt("my_prompt", version=1)

# Load a specific version of the prompt by URI
prompt = mlflow.genai.load_prompt("prompts:/my_prompt/1")

# Load a prompt version with an alias "production"
prompt = mlflow.genai.load_prompt("prompts:/my_prompt@production")

# Load the latest version of the prompt by URI
prompt = mlflow.genai.load_prompt("prompts:/my_prompt@latest")

# Load with custom cache TTL (5 minutes)
prompt = mlflow.genai.load_prompt("my_prompt", version=1, cache_ttl_seconds=300)

# Bypass cache entirely
prompt = mlflow.genai.load_prompt("my_prompt", version=1, cache_ttl_seconds=0)
```

mlflow.genai.set_prompt_alias(name="my_prompt", version=1, alias="production")

# Load the prompt by alias (use "@" to specify the alias)

# Delete the alias
mlflow.genai.delete_prompt_alias(name="my_prompt", alias="production")

## Model Configuration

### mlflow.genai.set_prompt_model_config

Set or update the model configuration for a specific prompt version.

```python
def set_prompt_model_config(
    name: str,
    version: str | int,
    model_config: PromptModelConfig | dict[str, Any],
) -> None
```

<ParamField path="name" type="str" required>
  The name of the prompt.
</ParamField>

<ParamField path="version" type="str | int" required>
  The version of the prompt.
</ParamField>

<ParamField path="model_config" type="PromptModelConfig | dict[str, Any]" required>
  A PromptModelConfig or dict with model settings like model_name, temperature.
</ParamField>

#### Example

```python
import mlflow
from mlflow.entities.model_registry import PromptModelConfig

# Set model config using a dictionary
mlflow.genai.set_prompt_model_config(
    name="my-prompt",
    version=1,
    model_config={"model_name": "gpt-4", "temperature": 0.7, "max_tokens": 1000},
)

# Set model config using PromptModelConfig for validation
config = PromptModelConfig(
    model_name="gpt-4-turbo",
    temperature=0.5,
    max_tokens=2000,
    top_p=0.95,
)
mlflow.genai.set_prompt_model_config(
    name="my-prompt",
    version=1,
    model_config=config,
)

# Load and verify the config was set
prompt = mlflow.genai.load_prompt("my-prompt", version=1)
print(prompt.model_config)
```

<Note>
Model configuration includes model-specific settings such as model name, temperature, max_tokens, and other inference parameters. Unlike the prompt template, model configuration is mutable and can be updated after a prompt version is created.
</Note>

## PromptVersion Class

### PromptVersion

An entity representing a specific version of a prompt with its template content.

```python
class PromptVersion:
    name: str
    version: int
    template: str | list[dict[str, Any]]
    response_format: dict[str, Any] | None
    model_config: dict[str, Any] | None
    commit_message: str | None
    creation_timestamp: int
    tags: dict[str, str]
    aliases: list[str]
```

#### Properties

<ParamField path="name" type="str">
  The name of the prompt.
</ParamField>

<ParamField path="version" type="int">
  The version number of the prompt.
</ParamField>

<ParamField path="template" type="str | list[dict[str, Any]]">
  The template content of the prompt. Can be either:
  - A string containing text with variables enclosed in double curly braces
  - A list of dictionaries representing chat messages
</ParamField>

<ParamField path="response_format" type="dict[str, Any] | None">
  Optional dictionary defining the expected response structure.
</ParamField>

<ParamField path="model_config" type="dict[str, Any] | None">
  Optional dictionary containing model-specific configuration.
</ParamField>

<ParamField path="commit_message" type="str | None">
  The commit message for the prompt version.
</ParamField>

<ParamField path="creation_timestamp" type="int">
  Timestamp of the prompt creation.
</ParamField>

<ParamField path="tags" type="dict[str, str]">
  A dictionary of tags associated with the prompt version.
</ParamField>

<ParamField path="aliases" type="list[str]">
  List of aliases for this prompt version.
</ParamField>

<ParamField path="variables" type="set[str]">
  A set of variables in the template text (enclosed in double curly braces).
</ParamField>

<ParamField path="uri" type="str">
  The URI of the prompt in the format `"prompts:/name/version"`.
</ParamField>

<ParamField path="is_text_prompt" type="bool">
  True if the prompt is a text prompt, False if it's a chat prompt.
</ParamField>

<ParamField path="is_chat_prompt" type="bool">
  True if the prompt is a chat prompt, False if it's a text prompt.
</ParamField>

#### Methods

**`format()`** - Format the template with the given keyword arguments.

```python
def format(
    self,
    allow_partial: bool = False,
    use_jinja_sandbox: bool = True,
    **kwargs,
) -> PromptVersion | str | list[dict[str, Any]]
```

<ParamField path="allow_partial" type="bool" default={false}>
  If True, allow partial formatting of the prompt text. If False, raise an error if there are missing variables.
</ParamField>

<ParamField path="use_jinja_sandbox" type="bool" default={true}>
  If True (default), use Jinja2's SandboxedEnvironment for safe rendering. Set to False to use unrestricted Environment. Only applies to Jinja2 templates (those containing `{% %}` syntax).
</ParamField>

<ParamField path="**kwargs">
  Keyword arguments to replace the variables in the template.
</ParamField>

<ResponseField name="return" type="PromptVersion | str | list[dict[str, Any]]">
  - For full formatting: returns the formatted string (text prompts) or list of messages (chat prompts)
  - For partial formatting: returns a new `PromptVersion` with the partially formatted template
</ResponseField>

**Example:**

```python
# Text prompt formatting
prompt = PromptVersion("my-prompt", 1, "Hello, {{title}} {{name}}!")
formatted = prompt.format(title="Ms", name="Alice")
print(formatted)
# Output: "Hello, Ms Alice!"

# Chat prompt formatting
chat_prompt = PromptVersion(
    "assistant",
    1,
    [
        {"role": "system", "content": "You are a {{style}} assistant."},
        {"role": "user", "content": "{{question}}"},
    ],
)
formatted = chat_prompt.format(style="friendly", question="How are you?")
print(formatted)
# Output: [{"role": "system", "content": "You are a friendly assistant."},
#          {"role": "user", "content": "How are you?"}]

# Partial formatting
formatted = prompt.format(title="Ms", allow_partial=True)
print(formatted)
# Output: PromptVersion(name=my-prompt, version=1, template="Hello, Ms {{name}}!")

# Jinja2 template formatting (with conditionals and loops)
jinja_prompt = PromptVersion(
    "jinja-prompt",
    1,
    "Hello {% if name %}{{ name }}{% else %}Guest{% endif %}!",
)
formatted = jinja_prompt.format(name="Alice")
print(formatted)
# Output: "Hello Alice!"
```

**`to_single_brace_format()`** - Convert the template to single brace format.

```python
def to_single_brace_format(self) -> str | list[dict[str, Any]]
```

<ResponseField name="return" type="str | list[dict[str, Any]]">
  The template with variables converted from `{{variable}}` to `{variable}` format.
  For text prompts, returns a string. For chat prompts, returns a list of messages.
</ResponseField>

This is useful for integrating with other systems that use single curly braces for variable replacement, such as LangChain's prompt template.

**Example:**

```python
prompt = mlflow.genai.load_prompt("my_prompt")
langchain_format = prompt.to_single_brace_format()
```

## PromptModelConfig Class

### PromptModelConfig

Configuration for a model associated with a prompt, including model name and inference parameters.

```python
class PromptModelConfig(BaseModel):
    provider: str | None = None
    model_name: str | None = None
    temperature: float | None = None
    max_tokens: int | None = None
    top_p: float | None = None
    top_k: int | None = None
    frequency_penalty: float | None = None
    presence_penalty: float | None = None
    stop_sequences: list[str] | None = None
    extra_params: dict[str, Any] = {}
```

<ParamField path="provider" type="str" optional>
  The model provider (e.g., "openai", "anthropic", "google").
</ParamField>

<ParamField path="model_name" type="str" optional>
  The name or identifier of the model (e.g., "gpt-4", "claude-3-opus").
</ParamField>

<ParamField path="temperature" type="float" optional>
  Sampling temperature for controlling randomness (typically 0.0-2.0). Lower values make output more deterministic, higher values more random.
</ParamField>

<ParamField path="max_tokens" type="int" optional>
  Maximum number of tokens to generate in the response.
</ParamField>

<ParamField path="top_p" type="float" optional>
  Nucleus sampling parameter (typically 0.0-1.0). The model considers tokens with top_p cumulative probability mass.
</ParamField>

<ParamField path="top_k" type="int" optional>
  Top-k sampling parameter. The model considers only the k most likely tokens.
</ParamField>

<ParamField path="frequency_penalty" type="float" optional>
  Penalty for token frequency (typically -2.0 to 2.0). Positive values reduce repetition of tokens based on their frequency in the text so far.
</ParamField>

<ParamField path="presence_penalty" type="float" optional>
  Penalty for token presence (typically -2.0 to 2.0). Positive values increase likelihood of introducing new topics.
</ParamField>

<ParamField path="stop_sequences" type="list[str]" optional>
  List of sequences that will cause the model to stop generating.
</ParamField>

<ParamField path="extra_params" type="dict[str, Any]" default={{}}>
  Additional model-specific parameters not covered by the standard fields. This allows for flexibility with provider-specific or experimental parameters.
</ParamField>

#### Example

```python
from mlflow.entities.model_registry import PromptModelConfig

# Basic configuration
config = PromptModelConfig(
    model_name="gpt-4",
    temperature=0.7,
    max_tokens=1000,
)

# Configuration with extra provider-specific params
config = PromptModelConfig(
    model_name="claude-3-opus",
    temperature=0.5,
    max_tokens=2000,
    extra_params={
        "anthropic_version": "2023-06-01",
        "response_metadata": {"cache_control": True},
    },
)

# Use with prompt registration
import mlflow

mlflow.genai.register_prompt(
    name="my_prompt",
    template="Analyze this: {{text}}",
    model_config=config,
)
```
