The MLflow Tracing API provides decorator-based and fluent APIs for instrumenting your GenAI applications and managing execution traces.

## Decorator API

### @mlflow.trace

Decorator that creates a new span for the decorated function.

<ParamField path="func" type="Callable">
  The function to be decorated. Must **not** be provided when using as a decorator.
</ParamField>

<ParamField path="name" type="str" optional>
  The name of the span. If not provided, the name of the function will be used.
</ParamField>

<ParamField path="span_type" type="str | SpanType" optional default="SpanType.UNKNOWN">
  The type of the span. Can be either a string or a `SpanType` enum value.
</ParamField>

<ParamField path="attributes" type="dict[str, Any]" optional>
  A dictionary of attributes to set on the span.
</ParamField>

<ParamField path="output_reducer" type="Callable[[list[Any]], Any]" optional>
  A function that reduces the outputs of the generator function into a single value to be set as the span output.
</ParamField>

<ParamField path="trace_destination" type="TraceLocationBase" optional>
  The destination to log the trace to, such as MLflow Experiment. If not provided, the destination will be an active MLflow experiment or a destination set by the `mlflow.tracing.set_destination` function. This parameter should only be used for root span and setting this for non-root spans will be ignored with a warning.
</ParamField>

<ParamField path="sampling_ratio_override" type="float" optional>
  The sampling ratio override for this specific function. Must be between 0.0 and 1.0. If provided, this overrides the global `MLFLOW_TRACE_SAMPLING_RATIO` setting. A value of 1.0 means all traces are sampled, 0.5 means 50% are sampled, and 0.0 means no traces are sampled. If not provided (None), the global sampling ratio is used. Note: This only applies to root spans; nested calls always trace if the parent is traced.
</ParamField>

<ResponseField name="return" type="Callable">
  The decorated function that will create a span when called.
</ResponseField>

#### Supported Function Types

| Function Type | Supported |
|---------------|-----------|
| Sync | ✅ |
| Async | ✅ (>= 2.16.0) |
| Generator | ✅ (>= 2.20.2) |
| Async Generator | ✅ (>= 2.20.2) |
| ClassMethod | ✅ (>= 3.0.0) |
| StaticMethod | ✅ (>= 3.0.0) |

#### Example

```python
import mlflow

@mlflow.trace
def my_function(x, y):
    return x + y

# Call the function - a trace will be automatically created
result = my_function(1, 2)

# Custom span name and type
@mlflow.trace(name="custom_span", span_type="LLM")
def llm_call(prompt):
    # Make LLM call
    return response

# Wrap external library functions
import math
mlflow.trace(math.factorial)(5)
```

## Fluent API

### mlflow.start_span

Context manager to create a new span and start it as the current span in the context.

```python
def start_span(
    name: str = "span",
    span_type: str | None = SpanType.UNKNOWN,
    attributes: dict[str, Any] | None = None,
    trace_destination: TraceLocationBase | None = None,
) -> Generator[LiveSpan, None, None]
```

<ParamField path="name" type="str" default="span">
  The name of the span.
</ParamField>

<ParamField path="span_type" type="str | SpanType" optional default="SpanType.UNKNOWN">
  The type of the span. Can be either a string or a `SpanType` enum value.
</ParamField>

<ParamField path="attributes" type="dict[str, Any]" optional>
  A dictionary of attributes to set on the span.
</ParamField>

<ParamField path="trace_destination" type="TraceLocationBase" optional>
  The destination to log the trace to, such as MLflow Experiment. If not provided, the destination will be an active MLflow experiment or a destination set by the `mlflow.tracing.set_destination` function. This parameter should only be used for root span and setting this for non-root spans will be ignored with a warning.
</ParamField>

<ResponseField name="yields" type="LiveSpan">
  An `mlflow.entities.Span` that represents the created span.
</ResponseField>

#### Example

```python
import mlflow

with mlflow.start_span("my_span") as span:
    x = 1
    y = 2
    span.set_inputs({"x": x, "y": y})
    
    z = x + y
    
    span.set_outputs(z)
    span.set_attribute("key", "value")
```

<Note>
When this context manager is used in the top-level scope (not within another span context), the span will be treated as a root span. The root span doesn't have a parent reference and **the entire trace will be logged when the root span is ended**.
</Note>

### mlflow.get_current_active_span

Get the current active span in the global context.

```python
def get_current_active_span() -> LiveSpan | None
```

<ResponseField name="return" type="LiveSpan | None">
  The current active span if exists, otherwise None.
</ResponseField>

#### Example

```python
import mlflow

@mlflow.trace
def f():
    span = mlflow.get_current_active_span()
    span.set_attribute("key", "value")
    return 0

f()
```

<Warning>
This only works when the span is created with fluent APIs like `@mlflow.trace` or `with mlflow.start_span`. If a span is created with the `mlflow.start_span_no_context` API, it won't be attached to the global context so this function will not return it.
</Warning>

## Trace Search and Management

### mlflow.search_traces

Return traces that match the given list of search expressions within the experiments.

```python
def search_traces(
    experiment_ids: list[str] | None = None,
    filter_string: str | None = None,
    max_results: int | None = None,
    order_by: list[str] | None = None,
    run_id: str | None = None,
    return_type: Literal["pandas", "list"] | None = None,
    model_id: str | None = None,
    sql_warehouse_id: str | None = None,
    include_spans: bool = True,
    locations: list[str] | None = None,
) -> pandas.DataFrame | list[Trace]
```

<ParamField path="experiment_ids" type="list[str]" optional>
  List of experiment ids to scope the search.
</ParamField>

<ParamField path="filter_string" type="str" optional>
  A search filter string.
</ParamField>

<ParamField path="max_results" type="int" optional>
  Maximum number of traces desired. If None, all traces matching the search expressions will be returned.
</ParamField>

<ParamField path="order_by" type="list[str]" optional>
  List of order_by clauses.
</ParamField>

<ParamField path="run_id" type="str" optional>
  A run id to scope the search. When a trace is created under an active run, it will be associated with the run and you can filter on the run id to retrieve the trace.
</ParamField>

<ParamField path="return_type" type="Literal['pandas', 'list']" optional>
  The type of the return value. If the pandas library is installed, the default return type is "pandas". Otherwise, the default return type is "list".
  
  - `"pandas"`: Returns a Pandas DataFrame containing information about traces where each row represents a single trace and each column represents a field of the trace (e.g., trace_id, spans, etc.)
  - `"list"`: Returns a list of `Trace` objects
</ParamField>

<ParamField path="model_id" type="str" optional>
  If specified, search traces associated with the given model ID.
</ParamField>

<ParamField path="sql_warehouse_id" type="str" optional>
  **DEPRECATED.** Use the `MLFLOW_TRACING_SQL_WAREHOUSE_ID` environment variable instead. The ID of the SQL warehouse to use for searching traces in inference tables or UC tables. Only used in Databricks.
</ParamField>

<ParamField path="include_spans" type="bool" default={true}>
  If `True`, include spans in the returned traces. Otherwise, only trace metadata is returned.
</ParamField>

<ParamField path="locations" type="list[str]" optional>
  List of trace location strings to scope the search.
</ParamField>

<ResponseField name="return" type="pandas.DataFrame | list[Trace]">
  Traces matching the search criteria, in the format specified by `return_type`.
</ResponseField>

#### Example

```python
import mlflow

# Search traces by experiment
traces_df = mlflow.search_traces(
    experiment_ids=["1"],
    filter_string="attributes.status = 'OK'",
    max_results=10
)

# Search traces by model ID
traces_df = mlflow.search_traces(model_id="m-074689226d3b40bfbbdf4c3ff35832cd")

# Get traces as a list of Trace objects
traces = mlflow.search_traces(
    experiment_ids=["1"],
    return_type="list"
)
```

<Note>
If expected number of search results is large, consider using the `MlflowClient.search_traces` API directly to paginate through the results. This function returns all results in memory and may not be suitable for large result sets.
</Note>

## Tracing Configuration

### mlflow.tracing.enable

Enable MLflow tracing globally.

```python
import mlflow

mlflow.tracing.enable()
```

### mlflow.tracing.disable

Disable MLflow tracing globally.

```python
import mlflow

mlflow.tracing.disable()
```

### mlflow.tracing.set_destination

Set the destination for logging traces.

```python
import mlflow

# Set destination to a specific experiment
mlflow.tracing.set_destination(experiment_id="123")
```

### mlflow.tracing.configure

Configure tracing settings including sampling ratio and other parameters.

```python
import mlflow

mlflow.tracing.configure(
    sampling_ratio=0.5,  # Sample 50% of traces
)
```
