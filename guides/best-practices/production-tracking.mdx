## Overview

Production tracking extends beyond training to monitor deployed models, track predictions, detect drift, and maintain system health. MLflow provides comprehensive tools for production observability.

## Production Run Tracking

### Create Dedicated Production Experiments

Separate production inference from training experiments:

```python
import mlflow

# Set up production experiment
mlflow.set_experiment("/production/fraud-detection-inference")

with mlflow.start_run(run_name="inference_batch_2024_03_15"):
    # Load production model
    model = mlflow.pyfunc.load_model("models:/fraud_detector@champion")
    
    # Run inference
    predictions = model.predict(batch_data)
    
    # Log production metrics
    mlflow.log_metrics({
        "batch_size": len(batch_data),
        "inference_time_ms": inference_time * 1000,
        "avg_prediction_confidence": predictions.mean(),
        "high_risk_predictions": (predictions > 0.8).sum()
    })
```

<Accordion title="Why separate production experiments">
Separate production experiments provide:
- Clear distinction between training and serving
- Focused monitoring and alerting
- Different access control and retention policies
- Easier audit trails for compliance
- Reduced noise in training experiment views
</Accordion>

## Feature Drift Detection

### Track Input Feature Statistics

Monitor feature distributions to detect data drift:

```python
import mlflow
import pandas as pd
import numpy as np

def log_feature_statistics(df, feature_columns):
    """Log comprehensive feature statistics"""
    stats = {}
    
    for col in feature_columns:
        if pd.api.types.is_numeric_dtype(df[col]):
            stats[f"{col}_mean"] = df[col].mean()
            stats[f"{col}_std"] = df[col].std()
            stats[f"{col}_min"] = df[col].min()
            stats[f"{col}_max"] = df[col].max()
            stats[f"{col}_missing_pct"] = df[col].isna().mean() * 100
        else:
            # Categorical features
            stats[f"{col}_unique_count"] = df[col].nunique()
            stats[f"{col}_missing_pct"] = df[col].isna().mean() * 100
    
    mlflow.log_metrics(stats)
    
    # Save detailed statistics
    detailed_stats = df[feature_columns].describe().to_dict()
    mlflow.log_dict(detailed_stats, "feature_statistics.json")

with mlflow.start_run():
    log_feature_statistics(production_data, feature_columns)
```

### Compare Against Training Baseline

Implement drift detection by comparing to training distribution:

```python
import mlflow
from scipy.stats import ks_2samp
import numpy as np

def detect_drift(training_data, production_data, features, threshold=0.05):
    """Detect drift using Kolmogorov-Smirnov test"""
    drift_results = {}
    
    for feature in features:
        if pd.api.types.is_numeric_dtype(training_data[feature]):
            # KS test for numeric features
            statistic, p_value = ks_2samp(
                training_data[feature].dropna(),
                production_data[feature].dropna()
            )
            
            drift_results[feature] = {
                "ks_statistic": statistic,
                "p_value": p_value,
                "drift_detected": p_value < threshold
            }
    
    return drift_results

with mlflow.start_run():
    drift_results = detect_drift(train_df, prod_df, feature_columns)
    
    # Log drift metrics
    for feature, result in drift_results.items():
        mlflow.log_metrics({
            f"{feature}_ks_statistic": result["ks_statistic"],
            f"{feature}_ks_pvalue": result["p_value"]
        })
    
    # Log drift summary
    drifted_features = [f for f, r in drift_results.items() if r["drift_detected"]]
    mlflow.log_metrics({
        "num_drifted_features": len(drifted_features),
        "drift_percentage": len(drifted_features) / len(drift_results) * 100
    })
    
    mlflow.log_dict(drift_results, "drift_analysis.json")
    
    if drifted_features:
        mlflow.set_tag("drift_alert", "true")
        mlflow.set_tag("drifted_features", ",".join(drifted_features))
```

<Accordion title="Drift detection strategies">
Common drift detection approaches:
- **Statistical tests**: KS test, Chi-square test, PSI (Population Stability Index)
- **Distance metrics**: KL divergence, Wasserstein distance
- **Model-based**: Monitor model confidence, prediction distribution
- **Threshold setting**: Use validation set to calibrate thresholds
- **Multi-level monitoring**: Track individual features and overall drift
</Accordion>

## Online and Real-time Tracking

### Batch Metrics for Real-time Serving

For high-throughput services, batch metrics before logging:

```python
import mlflow
import time
from collections import defaultdict
import threading

class MetricsBatcher:
    def __init__(self, flush_interval=60):
        self.metrics = defaultdict(list)
        self.flush_interval = flush_interval
        self.last_flush = time.time()
        self.lock = threading.Lock()
    
    def add_metric(self, key, value):
        with self.lock:
            self.metrics[key].append(value)
            
            if time.time() - self.last_flush > self.flush_interval:
                self.flush()
    
    def flush(self):
        with self.lock:
            if not self.metrics:
                return
            
            with mlflow.start_run():
                for key, values in self.metrics.items():
                    mlflow.log_metrics({
                        f"{key}_mean": sum(values) / len(values),
                        f"{key}_min": min(values),
                        f"{key}_max": max(values),
                        f"{key}_count": len(values)
                    })
            
            self.metrics.clear()
            self.last_flush = time.time()

# Usage in API endpoint
batcher = MetricsBatcher(flush_interval=60)

def predict_endpoint(data):
    start = time.time()
    prediction = model.predict(data)
    latency = time.time() - start
    
    batcher.add_metric("latency_ms", latency * 1000)
    batcher.add_metric("prediction_confidence", prediction.max())
    
    return prediction
```

### Use Async Logging for Performance

Enable async logging to avoid blocking production code:

```python
import mlflow
import os

# Enable async logging
os.environ["MLFLOW_ENABLE_ASYNC_LOGGING"] = "true"

with mlflow.start_run():
    # Non-blocking metric logging
    mlflow.log_metric("prediction_count", 1000)
    mlflow.log_params({"batch_id": "batch_001"})
    
    # Continue without waiting for logging to complete
    predictions = model.predict(data)

# Flush before shutdown
mlflow.flush_async_logging()
```

<Accordion title="Async logging best practices">
When using async logging:
- Always flush before application shutdown
- Set appropriate queue sizes for your workload
- Monitor for dropped metrics due to queue overflow
- Use for high-throughput scenarios (>1000 log operations/sec)
- Combine with batching for optimal performance
- Test failover behavior
</Accordion>

import mlflow
from mlflow.tracking import MlflowClient

def push_to_prometheus(metric_name, value, labels):

## Alerting and Anomaly Detection

### Implement Metric Thresholds

Set up alerts for metric degradation:

```python
import mlflow
from mlflow.tracking import MlflowClient

def check_metric_thresholds(run_id, thresholds):
    """Check if metrics meet threshold requirements"""
    client = MlflowClient()
    run = client.get_run(run_id)
    
    alerts = []
    for metric_name, threshold_config in thresholds.items():
        value = run.data.metrics.get(metric_name)
        if value is None:
            continue
        
        min_threshold = threshold_config.get("min")
        max_threshold = threshold_config.get("max")
        
        if min_threshold and value < min_threshold:
            alerts.append(f"{metric_name} ({value}) below threshold ({min_threshold})")
        if max_threshold and value > max_threshold:
            alerts.append(f"{metric_name} ({value}) above threshold ({max_threshold})")
    
    return alerts

thresholds = {
    "production_accuracy": {"min": 0.90},
    "latency_p99_ms": {"max": 100},
    "error_rate": {"max": 0.01}
}

with mlflow.start_run() as run:
    # Log metrics
    mlflow.log_metrics({...})
    
    # Check thresholds
    alerts = check_metric_thresholds(run.info.run_id, thresholds)
    
    if alerts:
        mlflow.set_tag("alerts", "; ".join(alerts))
        # Send notifications (email, Slack, PagerDuty, etc.)
        send_alert_notification(alerts)
```

## Summary

Effective production tracking in MLflow requires:
- Dedicated production experiments separate from training
- Comprehensive metric and prediction logging
- Feature drift detection and monitoring
- Performance and resource tracking
- A/B testing infrastructure
- Integration with external monitoring systems
- Alerting and anomaly detection
- Complete deployment metadata

By following these practices, you'll maintain full observability of production models and quickly detect issues before they impact business outcomes.
