Agentic AI systems—where multiple AI agents collaborate to accomplish complex tasks—require sophisticated monitoring. MLflow provides comprehensive observability for agent-based workflows, from simple tool-calling agents to complex multi-agent orchestrations.

## Understanding Agent Workflows

Agent workflows typically involve:
- **Multiple agents** with different roles and capabilities
- **Tool calling** where agents use external functions or APIs
- **Multi-step reasoning** with intermediate decisions
- **Dynamic execution paths** based on agent decisions
- **Collaboration** between agents to solve tasks

MLflow tracing captures all of this automatically.

## Quick Start: Monitoring a Simple Agent

<Steps>
  <Step title="Set up a basic LangChain agent">
    ```python
    import os
    from langchain.agents import AgentType, initialize_agent, load_tools
    from langchain.llms import OpenAI
    import mlflow

    # Ensure API keys are set
    assert "OPENAI_API_KEY" in os.environ
    assert "SERPAPI_API_KEY" in os.environ

    # Load LLM and tools
    llm = OpenAI(temperature=0)
    tools = load_tools(["serpapi", "llm-math"], llm=llm)

    # Initialize agent
    agent = initialize_agent(
        tools, 
        llm, 
        agent=AgentType.ZERO_SHOT_REACT_DESCRIPTION, 
        verbose=True
    )
    ```
  </Step>

  <Step title="Enable autologging">
    ```python
    # Enable LangChain autologging to capture agent traces
    mlflow.langchain.autolog(
        log_input_examples=True,
        log_model_signatures=True,
        log_models=True
    )
    ```
  </Step>

  <Step title="Run and monitor the agent">
    ```python
    with mlflow.start_run():
        # Log the agent
        logged_model = mlflow.langchain.log_model(agent, name="agent_model")

        # Run agent - all steps are traced
        question = "What was the high temperature in SF yesterday? What is that number raised to the 0.023 power?"
        
        # Load and use agent
        loaded_model = mlflow.pyfunc.load_model(logged_model.model_uri)
        answer = loaded_model.predict([{"input": question}])
        
        print(answer)
    ```
  </Step>

  <Step title="View agent execution in MLflow UI">
    Start the UI and navigate to the Traces tab to see:
    - Full agent reasoning chain
    - Tool calls and results
    - Decision-making steps
    - Timing for each operation
    ```bash
    mlflow ui
    ```
  </Step>
</Steps>

## Monitoring Multi-Agent Systems with CrewAI

For complex multi-agent orchestration:

```python
from textwrap import dedent
from crewai import Agent, Crew, Task
from crewai.knowledge.source.string_knowledge_source import StringKnowledgeSource
from crewai_tools import SerperDevTool, WebsiteSearchTool
import mlflow

mlflow.set_experiment("Multi-Agent-Trip-Planner")

# Enable CrewAI tracing
mlflow.crewai.autolog()

# Create knowledge source
content = "Users name is John. He is 30 years old and lives in San Francisco."
string_source = StringKnowledgeSource(
    content=content, 
    metadata={"preference": "personal"}
)

search_tool = SerperDevTool()
web_rag_tool = WebsiteSearchTool()

# Define agents with different roles
class TripAgents:
    def city_selection_agent(self):
        return Agent(
            role="City Selection Expert",
            goal="Select the best city based on weather, season, and prices",
            backstory="An expert in analyzing travel data to pick ideal destinations",
            tools=[search_tool, web_rag_tool],
            verbose=True,
        )

    def local_expert(self):
        return Agent(
            role="Local Expert at this city",
            goal="Provide the BEST insights about the selected city",
            backstory="""A knowledgeable local guide with extensive information
            about the city, it's attractions and customs""",
            tools=[search_tool, web_rag_tool],
            verbose=True,
        )

# Define tasks
class TripTasks:
    def identify_task(self, agent, origin, cities, interests, date_range):
        return Task(
            description=dedent(f"""
                Analyze and select the best city for the trip based
                on specific criteria such as weather patterns, seasonal
                events, and travel costs.
                
                Traveling from: {origin}
                City Options: {cities}
                Trip Date: {date_range}
                Traveler Interests: {interests}
            """),
            agent=agent,
            expected_output="Detailed report on the chosen city including flight costs, weather forecast, and attractions",
        )

    def gather_task(self, agent, origin, interests, date_range):
        return Task(
            description=dedent(f"""
                Compile an in-depth guide for someone traveling and wanting
                to have THE BEST trip ever! Find the best spots to go to,
                the kind of place only a local would know.
                
                Trip Date: {date_range}
                Traveling from: {origin}
                Traveler Interests: {interests}
            """),
            agent=agent,
            expected_output="Comprehensive city guide including hidden gems, cultural hotspots, and practical travel tips",
        )

# Create crew and run
class TripCrew:
    def __init__(self, origin, cities, date_range, interests):
        self.cities = cities
        self.origin = origin
        self.interests = interests
        self.date_range = date_range

    def run(self):
        agents = TripAgents()
        tasks = TripTasks()

        city_selector_agent = agents.city_selection_agent()
        local_expert_agent = agents.local_expert()

        identify_task = tasks.identify_task(
            city_selector_agent, self.origin, self.cities, 
            self.interests, self.date_range
        )
        gather_task = tasks.gather_task(
            local_expert_agent, self.origin, 
            self.interests, self.date_range
        )

        # Create crew with memory and knowledge
        crew = Crew(
            agents=[city_selector_agent, local_expert_agent],
            tasks=[identify_task, gather_task],
            verbose=True,
            memory=True,
            knowledge={
                "collection_name": "trip_planning",
                "sources": [string_source],
                "metadata": {"preference": "personal"},
            },
        )

        result = crew.kickoff()
        return result

# Run the multi-agent system - fully traced
trip_crew = TripCrew(
    "California", 
    "Tokyo", 
    "Dec 12 - Dec 20", 
    "sports"
)
result = trip_crew.run()
print(result)
```

MLflow will capture:
- Each agent's reasoning process
- Inter-agent communication
- Tool usage by each agent
- Memory operations
- Full execution timeline

## Monitoring PydanticAI Agents

For type-safe agent workflows:

```python
import mlflow
import mlflow.pydantic_ai
from dataclasses import dataclass
from pydantic import BaseModel, Field
from pydantic_ai import Agent, RunContext

mlflow.set_experiment("PydanticAI-Bank-Support")
mlflow.pydantic_ai.autolog()

# Define database (simulated)
class DatabaseConn:
    @classmethod
    async def customer_name(cls, *, id: int) -> str | None:
        if id == 123:
            return "John"

    @classmethod
    async def customer_balance(cls, *, id: int, include_pending: bool) -> float:
        if id == 123 and include_pending:
            return 123.45
        else:
            raise ValueError("Customer not found")

# Define dependencies and output schema
@dataclass
class SupportDependencies:
    customer_id: int
    db: DatabaseConn

class SupportOutput(BaseModel):
    support_advice: str = Field(description="Advice returned to the customer")
    block_card: bool = Field(description="Whether to block their card")
    risk: int = Field(description="Risk level of query", ge=0, le=10)

# Create agent with instrumentation
support_agent = Agent(
    "openai:gpt-4o",
    deps_type=SupportDependencies,
    output_type=SupportOutput,
    system_prompt=(
        "You are a support agent in our bank, give the "
        "customer support and judge the risk level of their query. "
        "Reply using the customer's name."
    ),
    instrument=True,  # Enable detailed instrumentation
)

@support_agent.system_prompt
async def add_customer_name(ctx: RunContext[SupportDependencies]) -> str:
    customer_name = await ctx.deps.db.customer_name(id=ctx.deps.customer_id)
    return f"The customer's name is {customer_name!r}"

@support_agent.tool
async def customer_balance(
    ctx: RunContext[SupportDependencies], 
    include_pending: bool
) -> str:
    """Returns the customer's current account balance."""
    balance = await ctx.deps.db.customer_balance(
        id=ctx.deps.customer_id,
        include_pending=include_pending,
    )
    return f"${balance:.2f}"

# Run agent - fully traced
deps = SupportDependencies(customer_id=123, db=DatabaseConn())

result = support_agent.run_sync("What is my balance?", deps=deps)
print(result.output)

result = support_agent.run_sync("I just lost my card!", deps=deps)
print(result.output)
```

## Key Metrics for Agent Monitoring

### Performance Metrics
- **Total execution time**: End-to-end workflow duration
- **Per-agent time**: Time spent in each agent
- **Tool call latency**: Time for external API/function calls
- **Token usage**: LLM tokens per agent and total

### Quality Metrics
- **Task success rate**: Percentage of completed tasks
- **Tool call accuracy**: Correct tool selection rate
- **Reasoning steps**: Number of steps to reach conclusion
- **Agent collaboration**: Inter-agent message count

### Cost Metrics
- **Token costs**: Cost breakdown by agent and model
- **Tool API costs**: External API usage costs
- **Total cost**: Complete workflow cost

## Debugging Agent Failures

When an agent workflow fails:

```python
import mlflow

try:
    with mlflow.start_run():
        result = agent.run(task)
except Exception as e:
    # Log the error with context
    mlflow.log_param("error_type", type(e).__name__)
    mlflow.log_param("error_message", str(e))
    
    # View trace in UI to see where it failed
    print(f"Check trace in MLflow UI for debugging: {mlflow.active_run().info.run_id}")
    raise
```

In the MLflow UI:
1. Navigate to the failed run
2. Open the Trace view
3. Identify the failing span (highlighted in red)
4. Examine inputs/outputs leading to failure
5. Check agent reasoning and tool calls

## Setting Up Alerts

Monitor agent performance in production:

```python
import mlflow

def monitor_agent_execution(agent, task, thresholds):
    """Monitor agent with alerting"""
    with mlflow.start_run():
        start = datetime.now()
        result = agent.run(task)
        duration = (datetime.now() - start).total_seconds()
        
        # Check thresholds
        if duration > thresholds["max_duration"]:
            alert(f"Agent execution exceeded {thresholds['max_duration']}s")
        
        # Log for analysis
        mlflow.log_metric("duration", duration)
        return result

result = monitor_agent_execution(
    agent, 
    task, 
    thresholds={"max_duration": 30}
)
```

## Best Practices

1. **Always enable autologging**: Capture complete execution traces
2. **Use experiments**: Organize agents by project or use case
3. **Tag runs**: Add metadata for easier filtering and search
4. **Set up baselines**: Track performance over time
5. **Monitor token usage**: Optimize costs without sacrificing quality
6. **Review failures**: Regularly analyze failed agent runs
7. **Version agents**: Track changes to agent configurations
