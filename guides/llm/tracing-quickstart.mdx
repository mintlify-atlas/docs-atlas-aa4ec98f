MLflow Tracing provides automatic observability for your LLM applications, capturing inputs, outputs, latency, and metadata across all model calls. This guide shows you how to enable tracing for popular LLM frameworks.

## Why Tracing?

Tracing helps you:
- Debug LLM application behavior
- Monitor performance and latency
- Understand token usage and costs
- Capture full conversation context
- Identify bottlenecks in multi-step workflows

## Quick Start with OpenAI

<Steps>
  <Step title="Install MLflow">
    ```bash
    pip install mlflow openai
    ```
  </Step>

  <Step title="Set your API key">
    ```bash
    export OPENAI_API_KEY="your-api-key"
    ```
  </Step>

  <Step title="Enable autologging and make calls">
    ```python
    import mlflow
    import openai

    # Enable automatic tracing
    mlflow.openai.autolog()

    client = openai.OpenAI()

    # All calls are automatically traced
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "system", "content": "You are a helpful assistant."},
            {"role": "user", "content": "What is MLflow?"}
        ]
    )

    print(response.choices[0].message.content)
    ```
  </Step>

  <Step title="View traces in MLflow UI">
    Start the MLflow UI to see your traces:
    ```bash
    mlflow ui
    ```
    Navigate to http://localhost:5000 and view the Traces tab.
  </Step>
</Steps>

## Tracing with Anthropic

MLflow supports automatic tracing for Anthropic's Claude models:

```python
import os
import anthropic
import mlflow

# Enable auto tracing for Anthropic
mlflow.anthropic.autolog()

client = anthropic.Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

# Use the create method - automatically traced
message = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    max_tokens=1024,
    messages=[
        {"role": "user", "content": "Hello, Claude"},
    ],
)
print(message.content)
```

## Tracing with LangChain

For LangChain applications, enable autologging to capture the entire chain execution:

```python
import os
from langchain.llms import OpenAI
from langchain.prompts import PromptTemplate
from langchain.schema.output_parser import StrOutputParser
import mlflow

assert "OPENAI_API_KEY" in os.environ

# Enable LangChain autologging
mlflow.langchain.autolog(
    log_input_examples=True,
    log_model_signatures=True,
    log_models=True
)

prompt = PromptTemplate(
    input_variables=["question"],
    template="Answer the following question: {question}"
)

llm = OpenAI(temperature=0.9)
chain = prompt | llm | StrOutputParser()

# Chain invocation is automatically traced
result = chain.invoke({"question": "Who owns MLflow?")
print(result)
```

## Tracing Multi-Agent Workflows

MLflow also traces complex multi-agent frameworks like CrewAI:

```python
from crewai import Agent, Crew, Task
import mlflow

mlflow.set_experiment("CrewAI-Demo")

# Enable CrewAI tracing
mlflow.crewai.autolog()

# Define agents
researcher = Agent(
    role="Research Analyst",
    goal="Research and analyze information",
    backstory="Expert researcher with attention to detail",
    verbose=True,
)

writer = Agent(
    role="Content Writer",
    goal="Create engaging content",
    backstory="Skilled writer with storytelling expertise",
    verbose=True,
)

# Define tasks
research_task = Task(
    description="Research the latest trends in AI",
    agent=researcher,
    expected_output="Detailed research report"
)

writing_task = Task(
    description="Write a blog post based on the research",
    agent=writer,
    expected_output="Engaging blog post"
)

# Create and run crew - full trace captured
crew = Crew(
    agents=[researcher, writer],
    tasks=[research_task, writing_task],
    verbose=True
)

result = crew.kickoff()
print(result)
```

## Supported Frameworks

MLflow provides automatic tracing for:

- **OpenAI** - `mlflow.openai.autolog()`
- **Anthropic** - `mlflow.anthropic.autolog()`
- **LangChain** - `mlflow.langchain.autolog()`
- **CrewAI** - `mlflow.crewai.autolog()`
- **PydanticAI** - `mlflow.pydantic_ai.autolog()`
- **Haystack** - `mlflow.haystack.autolog()`
- **Gemini** - `mlflow.gemini.autolog()`
- **Groq** - `mlflow.groq.autolog()`
- **Mistral** - `mlflow.mistral.autolog()`

## Manual Tracing

For custom workflows or unsupported frameworks, use manual tracing:

```python
import mlflow

with mlflow.start_span(name="custom_llm_call") as span:
    # Your LLM call
    response = your_custom_llm_function()
    
    # Log additional metadata
    span.set_inputs({"prompt": "your prompt"})
    span.set_outputs({"response": response})
    span.set_attributes({
        "model": "custom-model",
        "temperature": 0.7
    })
```

## Viewing Traces

Once tracing is enabled:

1. Start MLflow UI: `mlflow ui`
2. Navigate to the **Traces** tab
3. Click on a trace to see:
   - Full input/output data
   - Token usage and costs
   - Latency breakdown
   - Nested spans for multi-step workflows
   - Model parameters and metadata
