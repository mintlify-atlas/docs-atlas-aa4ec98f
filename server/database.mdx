## Overview

MLflow uses a backend store to persist experiment and run metadata. While SQLite is suitable for development, production deployments should use a database server like PostgreSQL or MySQL.

## Supported Databases

MLflow supports the following databases through SQLAlchemy:

- **SQLite** - File-based database (development only)
- **PostgreSQL** - Recommended for production
- **MySQL** - Alternative for production
- **MSSQL** - Microsoft SQL Server

<Note>
PostgreSQL is the recommended database for production deployments due to its reliability, performance, and feature set.
</Note>

## SQLite Setup

### Local File Database

SQLite is the default backend and requires no additional setup:

```bash
# Automatically creates mlflow.db
mlflow server --backend-store-uri sqlite:///mlflow.db
```

<CodeGroup>
```bash Relative Path
mlflow server --backend-store-uri sqlite:///mlflow.db
```

```bash Absolute Path
mlflow server --backend-store-uri sqlite:////data/mlflow/mlflow.db
```

```bash In-Memory (Testing Only)
mlflow server --backend-store-uri sqlite:///:memory:
```
</CodeGroup>

<Warning>
SQLite is **not recommended for production** due to:
- No concurrent write support
- Limited scalability
- No built-in replication
- Performance issues with large datasets
</Warning>

## PostgreSQL Setup

### Installation

<Steps>
  <Step title="Install PostgreSQL">
    <CodeGroup>
    ```bash Ubuntu/Debian
    sudo apt update
    sudo apt install postgresql postgresql-contrib
    ```

    ```bash macOS
    brew install postgresql@15
    brew services start postgresql@15
    ```

    ```bash Docker
    docker run -d \
      --name mlflow-postgres \
      -e POSTGRES_USER=mlflow \
      -e POSTGRES_PASSWORD=mlflow \
      -e POSTGRES_DB=mlflow \
      -p 5432:5432 \
      postgres:15
    ```
    </CodeGroup>
  </Step>

  <Step title="Create database and user">
    Connect to PostgreSQL:

    ```bash
    sudo -u postgres psql
    ```

    Create database and user:

    ```sql
    CREATE DATABASE mlflow;
    CREATE USER mlflow WITH ENCRYPTED PASSWORD 'secure_password';
    GRANT ALL PRIVILEGES ON DATABASE mlflow TO mlflow;
    
    -- For PostgreSQL 15+, also grant schema privileges
    \c mlflow
    GRANT ALL ON SCHEMA public TO mlflow;
    ```
  </Step>

  <Step title="Install Python driver">
    ```bash
    pip install psycopg2-binary
    ```
  </Step>

  <Step title="Configure MLflow server">
    ```bash
    mlflow server \
      --backend-store-uri postgresql://mlflow:secure_password@localhost:5432/mlflow \
      --default-artifact-root ./mlruns
    ```
  </Step>
</Steps>

### Connection String Format

```
postgresql://[user]:[password]@[host]:[port]/[database]
```

**Examples:**

<CodeGroup>
```bash Local Server
postgresql://mlflow:password@localhost:5432/mlflow
```

```bash Remote Server
postgresql://mlflow:password@db.company.com:5432/mlflow
```

```bash With SSL
postgresql://mlflow:password@db.company.com:5432/mlflow?sslmode=require
```

```bash AWS RDS
postgresql://mlflow:password@mlflow-db.abc123.us-east-1.rds.amazonaws.com:5432/mlflow
```
</CodeGroup>

### PostgreSQL Optimization

Optimal PostgreSQL configuration for MLflow:

```sql postgresql.conf
# Memory settings
shared_buffers = 4GB
effective_cache_size = 12GB
maintenance_work_mem = 1GB
work_mem = 64MB

# Connection settings
max_connections = 200

# Query tuning
random_page_cost = 1.1  # For SSD storage
effective_io_concurrency = 200

# WAL settings
wal_buffers = 16MB
checkpoint_completion_target = 0.9
```

### Creating Indexes

Improve query performance with additional indexes:

```sql
-- Connect to the database
\c mlflow

-- Index for experiment searches
CREATE INDEX idx_experiments_name ON experiments(name);
CREATE INDEX idx_experiments_lifecycle_stage ON experiments(lifecycle_stage);

-- Index for run searches
CREATE INDEX idx_runs_experiment_id ON runs(experiment_id);
CREATE INDEX idx_runs_start_time ON runs(start_time DESC);
CREATE INDEX idx_runs_user_id ON runs(user_id);

-- Index for metrics
CREATE INDEX idx_metrics_run_id ON metrics(run_uuid);
CREATE INDEX idx_metrics_key ON metrics(key);

-- Index for parameters
CREATE INDEX idx_params_run_id ON params(run_uuid);
CREATE INDEX idx_params_key ON params(key);

-- Index for tags
CREATE INDEX idx_tags_run_id ON tags(run_uuid);
CREATE INDEX idx_tags_key ON tags(key);
```

## MySQL Setup

### Installation

<Steps>
  <Step title="Install MySQL">
    <CodeGroup>
    ```bash Ubuntu/Debian
    sudo apt update
    sudo apt install mysql-server
    sudo mysql_secure_installation
    ```

    ```bash macOS
    brew install mysql
    brew services start mysql
    ```

    ```bash Docker
    docker run -d \
      --name mlflow-mysql \
      -e MYSQL_ROOT_PASSWORD=rootpassword \
      -e MYSQL_DATABASE=mlflow \
      -e MYSQL_USER=mlflow \
      -e MYSQL_PASSWORD=mlflow \
      -p 3306:3306 \
      mysql:8.0
    ```
    </CodeGroup>
  </Step>

  <Step title="Create database and user">
    Connect to MySQL:

    ```bash
    sudo mysql -u root -p
    ```

    Create database and user:

    ```sql
    CREATE DATABASE mlflow CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
    CREATE USER 'mlflow'@'%' IDENTIFIED BY 'secure_password';
    GRANT ALL PRIVILEGES ON mlflow.* TO 'mlflow'@'%';
    FLUSH PRIVILEGES;
    ```
  </Step>

  <Step title="Install Python driver">
    ```bash
    pip install pymysql
    ```
  </Step>

  <Step title="Configure MLflow server">
    ```bash
    mlflow server \
      --backend-store-uri mysql+pymysql://mlflow:secure_password@localhost:3306/mlflow \
      --default-artifact-root ./mlruns
    ```
  </Step>
</Steps>

### Connection String Format

```
mysql+pymysql://[user]:[password]@[host]:[port]/[database]
```

**Examples:**

<CodeGroup>
```bash Local Server
mysql+pymysql://mlflow:password@localhost:3306/mlflow
```

```bash Remote Server
mysql+pymysql://mlflow:password@db.company.com:3306/mlflow
```

```bash AWS RDS
mysql+pymysql://mlflow:password@mlflow-db.abc123.us-east-1.rds.amazonaws.com:3306/mlflow
```

```bash With SSL
mysql+pymysql://mlflow:password@db.company.com:3306/mlflow?ssl_ca=/path/to/ca.pem
```
</CodeGroup>

### MySQL Optimization

```sql my.cnf
[mysqld]
# InnoDB settings
innodb_buffer_pool_size = 4G
innodb_log_file_size = 512M
innodb_flush_log_at_trx_commit = 2
innodb_flush_method = O_DIRECT

# Connection settings
max_connections = 200

# Query cache (MySQL 5.7 and earlier)
query_cache_type = 1
query_cache_size = 256M

# Character set
character_set_server = utf8mb4
collation_server = utf8mb4_unicode_ci
```

## Database Migration

### Initial Schema Creation

MLflow automatically creates tables on first run. To manually initialize:

```bash
mlflow db upgrade <database-uri>
```

**Examples:**

<CodeGroup>
```bash PostgreSQL
mlflow db upgrade postgresql://mlflow:password@localhost:5432/mlflow
```

```bash MySQL
mlflow db upgrade mysql+pymysql://mlflow:password@localhost:3306/mlflow
```

```bash SQLite
mlflow db upgrade sqlite:///mlflow.db
```
</CodeGroup>

### Upgrading Database Schema

Before upgrading MLflow, upgrade the database schema:

<Steps>
  <Step title="Backup your database">
    <CodeGroup>
    ```bash PostgreSQL
    pg_dump -U mlflow -h localhost mlflow > mlflow_backup.sql
    ```

    ```bash MySQL
    mysqldump -u mlflow -p mlflow > mlflow_backup.sql
    ```

    ```bash SQLite
    cp mlflow.db mlflow.db.backup
    ```
    </CodeGroup>
  </Step>

  <Step title="Upgrade schema">
    ```bash
    mlflow db upgrade postgresql://mlflow:password@localhost:5432/mlflow
    ```

    <Note>
    Schema migrations can be slow. Review the [migrations README](https://github.com/mlflow/mlflow/blob/master/mlflow/store/db_migrations/README.md) for performance estimates.
    </Note>
  </Step>

  <Step title="Verify migration">
    Start the MLflow server and verify functionality:

    ```bash
    mlflow server --backend-store-uri postgresql://mlflow:password@localhost:5432/mlflow
    ```
  </Step>
</Steps>

<Warning>
Always backup your database before running migrations. Schema migrations are not guaranteed to be transactional.
</Warning>


## High Availability Setup

### PostgreSQL with Replication

<Steps>
  <Step title="Set up primary database">
    Configure PostgreSQL primary server for replication in `postgresql.conf`:

    ```sql postgresql.conf
    wal_level = replica
    max_wal_senders = 3
    wal_keep_size = 64
    ```
  </Step>

  <Step title="Create replication user">
    ```sql
    CREATE USER replicator WITH REPLICATION ENCRYPTED PASSWORD 'repl_password';
    ```
  </Step>

  <Step title="Configure standby server">
    On standby server, create `standby.signal` file and configure:

    ```sql postgresql.auto.conf
    primary_conninfo = 'host=primary-server port=5432 user=replicator password=repl_password'
    ```
  </Step>

  <Step title="Use connection pooler">
    Use PgBouncer or similar for connection pooling:

    ```bash
    mlflow server \
      --backend-store-uri postgresql://mlflow:password@pgbouncer:6432/mlflow
    ```
  </Step>
</Steps>

### Load Balancing

For read-heavy workloads, use read replicas:

```bash
# Primary for writes
export PRIMARY_DB=postgresql://mlflow:password@primary:5432/mlflow

# Replica for reads (if supported by your deployment)
export REPLICA_DB=postgresql://mlflow:password@replica:5432/mlflow

mlflow server --backend-store-uri $PRIMARY_DB
```

    ```bash
      --db-instance-identifier mlflow-db \
      --db-instance-class db.t3.medium \

      --engine-version 15.3 \
    ```

    ```bash
      --query 'DBInstances[0].Endpoint.Address' \
    ```

  <Step title="Connect MLflow">

  </Step>
</Steps>

## Database Maintenance

### Vacuum and Analyze (PostgreSQL)

Regular maintenance improves query performance:

```sql
-- Full vacuum and analyze
VACUUM ANALYZE;

-- Vacuum specific tables
VACUUM ANALYZE experiments;
VACUUM ANALYZE runs;
VACUUM ANALYZE metrics;
```

Automate with cron:

```bash
0 2 * * 0 psql -U mlflow -d mlflow -c "VACUUM ANALYZE;"
```

### Optimize Tables (MySQL)

```sql
-- Optimize all tables
OPTIMIZE TABLE experiments, runs, metrics, params, tags;

-- Analyze for better query plans
ANALYZE TABLE experiments, runs, metrics, params, tags;
```

### Backup Strategies

<CodeGroup>
```bash PostgreSQL Backup
# Full backup
pg_dump -U mlflow -h localhost mlflow | gzip > mlflow_$(date +%Y%m%d).sql.gz

# Restore
gunzip < mlflow_20240101.sql.gz | psql -U mlflow -d mlflow
```

```bash MySQL Backup
# Full backup
mysqldump -u mlflow -p mlflow | gzip > mlflow_$(date +%Y%m%d).sql.gz

# Restore
gunzip < mlflow_20240101.sql.gz | mysql -u mlflow -p mlflow
```

```bash Automated Backup Script
#!/bin/bash
BACKUP_DIR="/backups/mlflow"
DATE=$(date +%Y%m%d_%H%M%S)

# Create backup
pg_dump -U mlflow mlflow | gzip > "$BACKUP_DIR/mlflow_$DATE.sql.gz"

# Keep only last 30 days
find "$BACKUP_DIR" -name "mlflow_*.sql.gz" -mtime +30 -delete
```
</CodeGroup>

## Monitoring Database Performance

### PostgreSQL Monitoring

```sql
-- Check active connections
SELECT count(*) FROM pg_stat_activity;

-- Check slow queries
SELECT pid, now() - pg_stat_activity.query_start AS duration, query
FROM pg_stat_activity
WHERE state = 'active'
ORDER BY duration DESC;

-- Check table sizes
SELECT tablename, pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename)) AS size
FROM pg_tables
WHERE schemaname = 'public'
ORDER BY pg_total_relation_size(schemaname||'.'||tablename) DESC;

-- Check index usage
SELECT schemaname, tablename, indexname, idx_scan
FROM pg_stat_user_indexes
ORDER BY idx_scan ASC;
```

### MySQL Monitoring

```sql
-- Check active connections
SHOW PROCESSLIST;

-- Check slow queries (enable slow query log first)
SHOW VARIABLES LIKE 'slow_query_log';
SET GLOBAL slow_query_log = 'ON';

-- Check table sizes
SELECT table_name, 
       ROUND(((data_length + index_length) / 1024 / 1024), 2) AS size_mb
FROM information_schema.TABLES
WHERE table_schema = 'mlflow'
ORDER BY size_mb DESC;
```

## Troubleshooting

### Connection Timeout

```bash
# Test database connectivity
telnet db-server 5432  # PostgreSQL
telnet db-server 3306  # MySQL

# Test credentials
psql -h db-server -U mlflow -d mlflow  # PostgreSQL
mysql -h db-server -u mlflow -p mlflow  # MySQL
```

### Too Many Connections

```sql
-- PostgreSQL: Increase max connections
ALTER SYSTEM SET max_connections = 300;
SELECT pg_reload_conf();

-- MySQL: Increase max connections
SET GLOBAL max_connections = 300;
```

### Slow Queries

Enable query logging:

```sql
-- PostgreSQL
ALTER SYSTEM SET log_min_duration_statement = 1000;  -- Log queries > 1s
SELECT pg_reload_conf();

-- MySQL
SET GLOBAL slow_query_log = 'ON';
SET GLOBAL long_query_time = 1;
```

### Database Locked (SQLite)

SQLite doesn't support concurrent writes:

```bash
# Use a proper database server for production!
mlflow server --backend-store-uri postgresql://...
```
