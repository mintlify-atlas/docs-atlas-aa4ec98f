## What is a Run?

A run is a single execution of machine learning code. It's the fundamental unit of organization in MLflow, representing one iteration of model training, evaluation, or any other ML task you want to track.

Each run records:
- Parameters (input configuration)
- Metrics (output measurements)
- Tags (metadata)
- Artifacts (output files)
- Source code version and entry point
- Start and end times
- Execution status

## Run Structure

A `Run` object consists of two main components:

### RunInfo - Metadata

Contains metadata about the run execution:

```python
from mlflow.entities import Run

# Access run information
run.info.run_id              # Unique run identifier
run.info.experiment_id       # Parent experiment ID
run.info.run_name           # Human-readable name
run.info.user_id            # User who created the run
run.info.status             # RUNNING, FINISHED, FAILED, KILLED
run.info.start_time         # Start timestamp (milliseconds)
run.info.end_time           # End timestamp (milliseconds)
run.info.artifact_uri       # Location of artifacts
run.info.lifecycle_stage    # 'active' or 'deleted'
```

## Starting and Managing Runs

### Basic Run Lifecycle

```python
import mlflow

# Method 1: Context manager (recommended)
with mlflow.start_run(run_name="training-run-1") as run:
    mlflow.log_param("learning_rate", 0.01)
    mlflow.log_metric("accuracy", 0.95)
    print(f"Run ID: {run.info.run_id}")
# Run automatically ends when exiting the context

# Method 2: Manual management
run = mlflow.start_run(run_name="training-run-2")
try:
    mlflow.log_param("epochs", 100)
    mlflow.log_metric("loss", 0.05)
finally:
    mlflow.end_run()  # Always end the run
```

### Nested Runs

Create hierarchical run structures for complex workflows:

```python
import mlflow

# Parent run for hyperparameter search
with mlflow.start_run(run_name="hyperparameter-search") as parent_run:
    mlflow.log_param("search_type", "grid")
    
    # Child runs for individual configurations
    for lr in [0.001, 0.01, 0.1]:
        with mlflow.start_run(run_name=f"lr-{lr}", nested=True) as child_run:
            mlflow.log_param("learning_rate", lr)
            
            # Train and evaluate
            accuracy = train_model(lr)
            mlflow.log_metric("accuracy", accuracy)
            
            print(f"Child run {child_run.info.run_id} under parent {parent_run.info.run_id}")
```

### Getting Parent Runs

```python
import mlflow

with mlflow.start_run(run_name="parent") as parent:
    with mlflow.start_run(run_name="child", nested=True) as child:
        # Get the parent run from within a nested run
        parent_run = mlflow.get_parent_run(child.info.run_id)
        print(f"Parent: {parent_run.info.run_name}")
```

## Run Status

Runs can have different statuses throughout their lifecycle:

```python
from mlflow.entities import RunStatus

# Possible run statuses
RunStatus.RUNNING    # Run is currently executing
RunStatus.SCHEDULED  # Run is scheduled but not started
RunStatus.FINISHED   # Run completed successfully
RunStatus.FAILED     # Run failed with an error
RunStatus.KILLED     # Run was manually terminated
```

## Accessing Runs

### Get Active Run

```python
import mlflow

with mlflow.start_run():
```

```python
        "criterion": "gini"
```

### Metrics

Metrics are numeric values that can be updated over time:

```python
import mlflow

with mlflow.start_run():
    # Log metrics with steps for tracking progress
    for epoch in range(100):
        train_loss = train_epoch()
        val_loss = validate()
        
        mlflow.log_metric("train_loss", train_loss, step=epoch)
        mlflow.log_metric("val_loss", val_loss, step=epoch)
    
    # Log final metrics
    mlflow.log_metrics({
        "final_accuracy": 0.95,
        "final_f1": 0.93
    })
```

### Tags

Tags are key-value metadata for organizing and filtering runs:

```python

with mlflow.start_run():

        "dataset_version": "v2.1",

        "feature_selection": "true"
```

### Artifacts

Log files, models, and other outputs:

```python
import mlflow
import pandas as pd

with mlflow.start_run():
    # Log files
    mlflow.log_artifact("trained_model.pkl")
    mlflow.log_artifacts("output_directory/")
    
    # Log tables
    df = pd.DataFrame({"feature": [1, 2], "importance": [0.8, 0.2]})
    mlflow.log_table(df, "feature_importance.json")
    
    # Log images
    import matplotlib.pyplot as plt
    fig, ax = plt.subplots()
    ax.plot([1, 2, 3], [1, 4, 9])
    mlflow.log_figure(fig, "training_curve.png")
```

## Searching Runs

### Filter and Order Runs

```python
import mlflow

# Search with filters
runs = mlflow.search_runs(
    experiment_ids=["1", "2"],
    filter_string="metrics.accuracy > 0.9 AND params.model_type = 'cnn'",
    order_by=["metrics.accuracy DESC"],
    max_results=20
)

for _, run in runs.iterrows():
    print(f"Run {run.run_id}: accuracy = {run['metrics.accuracy']}")
```

### Advanced Search Queries

```python
import mlflow
from mlflow.entities import ViewType

# Search with complex filters
runs = mlflow.search_runs(
    filter_string="""
        metrics.f1_score > 0.85 AND
        params.framework = 'pytorch' AND
        tags.stage = 'production'
    """,
    order_by=["start_time DESC"],
    max_results=50,
    run_view_type=ViewType.ACTIVE_ONLY
)
```

## Best Practices

<AccordionGroup>
  <Accordion title="Always Use Context Managers">
    Use `with mlflow.start_run()` to ensure runs are properly closed even if exceptions occur:
    
    ```python
    # Good
    with mlflow.start_run():
        train_model()
    
    # Avoid
    mlflow.start_run()
    train_model()
    mlflow.end_run()  # May not be called if exception occurs
    ```
  </Accordion>
  
  <Accordion title="Set Descriptive Run Names">
    Use meaningful names that describe what the run does:
    
    ```python
    # Good
    with mlflow.start_run(run_name="resnet50-imagenet-lr0.01"):
        ...
    
    # Avoid
    with mlflow.start_run(run_name="run1"):
        ...
    ```
  </Accordion>
  
  <Accordion title="Log Parameters First">
    Log all parameters at the start of your run before training begins. This helps with debugging if runs fail.
  </Accordion>
  
  <Accordion title="Use Steps for Metrics">
    Always log metrics with step numbers to track progress over time:
    
    ```python
    for epoch in range(num_epochs):
        loss = train_epoch()
        mlflow.log_metric("loss", loss, step=epoch)
    ```
  </Accordion>
  
  <Accordion title="Add Context with Tags">
    Use tags to add searchable metadata:
    
    ```python
    mlflow.set_tags({
        "git_commit": "abc123",
        "dataset_version": "v2.0",
        "environment": "production"
    })
    ```
  </Accordion>
</AccordionGroup>
