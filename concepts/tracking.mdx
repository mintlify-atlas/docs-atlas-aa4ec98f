## What is MLflow Tracking?

MLflow Tracking is a component that provides APIs for logging parameters, code versions, metrics, and artifacts when running machine learning code. It allows you to query and compare results across different runs and experiments.

The tracking system is built around a client-server architecture where the tracking server stores run data and artifacts, which can be queried through the MLflow API.

## Core Components

### Tracking URI

The tracking URI determines where MLflow logs run data. You can set it using:

```python
import mlflow

# Set tracking URI to a local directory
mlflow.set_tracking_uri("file:///path/to/mlruns")

# Set tracking URI to a remote server
mlflow.set_tracking_uri("http://localhost:5000")

# Set tracking URI to Databricks
mlflow.set_tracking_uri("databricks")

# Get current tracking URI
uri = mlflow.get_tracking_uri()
print(f"Current tracking URI: {uri}")
```

### Tracking Service Client

The `TrackingServiceClient` is a lower-level API that directly translates to MLflow REST API calls. For most use cases, the fluent API (top-level `mlflow` module functions) is recommended.

```python
from mlflow.tracking._tracking_service.client import TrackingServiceClient
from mlflow.tracking._tracking_service.utils import _resolve_tracking_uri

client = TrackingServiceClient(_resolve_tracking_uri())

# Get experiment by name
experiment = client.get_experiment_by_name("my-experiment")
```

## Fluent API

MLflow provides a high-level "fluent" API for managing experiments and runs. This is the most common way to interact with the tracking system.

### Basic Tracking Workflow

```python
import mlflow

# Set the active experiment
mlflow.set_experiment("my-experiment")

# Start a run
with mlflow.start_run() as run:
    # Log parameters
    mlflow.log_param("learning_rate", 0.01)
    mlflow.log_param("batch_size", 32)
    
    # Log metrics
    mlflow.log_metric("accuracy", 0.95)
    mlflow.log_metric("loss", 0.05)
    
    # Log artifacts
    mlflow.log_artifact("model.pkl")
    
    print(f"Run ID: {run.info.run_id}")
```

### Managing Active Runs

MLflow maintains an active run stack to support nested runs:

```python
import mlflow

# Get the currently active run
active_run = mlflow.active_run()
if active_run:
    print(f"Active run ID: {active_run.info.run_id}")

# Manual run management (without context manager)
mlflow.start_run()
mlflow.log_param("alpha", 0.5)
mlflow.end_run()

# Nested runs for hierarchical experiments
with mlflow.start_run(run_name="parent") as parent:
    mlflow.log_param("parent_param", "value")
    
    with mlflow.start_run(run_name="child", nested=True) as child:
        mlflow.log_param("child_param", "value")
        print(f"Parent ID: {parent.info.run_id}")
        print(f"Child ID: {child.info.run_id}")
```

## Logging Functions

### Parameters

Parameters are key-value pairs that describe input settings for your run. They are typically configuration values like hyperparameters.

```python
import mlflow

with mlflow.start_run():
    # Log single parameter
    mlflow.log_param("epochs", 100)
    
    # Log multiple parameters
    mlflow.log_params({
        "learning_rate": 0.001,
        "optimizer": "adam",
        "hidden_layers": 3
    })
```

### Metrics

Metrics are numeric values that can be logged multiple times during a run. They typically represent model performance.

```python
import mlflow

with mlflow.start_run():
    # Log single metric
    mlflow.log_metric("mse", 0.25, step=1)
    
    # Log metrics over multiple steps
    for epoch in range(10):
        loss = train_epoch()  # Your training function
        mlflow.log_metric("loss", loss, step=epoch)
    
    # Log multiple metrics at once
    mlflow.log_metrics({
        "train_acc": 0.92,
        "val_acc": 0.89,
        "test_acc": 0.87
    })
```

### Artifacts

Artifacts are files associated with a run, such as models, images, or data files.

```python
import mlflow
import matplotlib.pyplot as plt

with mlflow.start_run():
    # Log a single file
    mlflow.log_artifact("output.txt")
    
    # Log an entire directory
    mlflow.log_artifacts("model_checkpoints/")
    
    # Log a figure
    fig, ax = plt.subplots()
    ax.plot([1, 2, 3], [4, 5, 6])
    mlflow.log_figure(fig, "plot.png")
    
    # Log an image
    from PIL import Image
    image = Image.new("RGB", (100, 100))
    mlflow.log_image(image, "sample.png")
    
    # Log a dictionary as JSON
    mlflow.log_dict({"config": "value"}, "config.json")
    
    # Log text content
    mlflow.log_text("Some text content", "notes.txt")
```

## Searching and Querying Runs

```python
import mlflow
from mlflow.entities import ViewType

# Search runs with filter
runs = mlflow.search_runs(
    experiment_ids=["1"],
    filter_string="metrics.accuracy > 0.9",
    order_by=["metrics.accuracy DESC"],
    max_results=10
)

# Search across all experiments
all_runs = mlflow.search_runs(
    filter_string="params.model_type = 'neural_network'"
)

# Include deleted runs
all_runs_including_deleted = mlflow.search_runs(
    filter_string="",
    run_view_type=ViewType.ALL
)

# Get a specific run
run = mlflow.get_run("run_id_here")
print(f"Run name: {run.info.run_name}")
print(f"Status: {run.info.status}")
print(f"Metrics: {run.data.metrics}")
print(f"Parameters: {run.data.params}")
```

## Artifact Management

Retrieve artifact URIs and download artifacts:

```python
import mlflow

with mlflow.start_run() as run:
    mlflow.log_artifact("model.pkl")
    
    # Get artifact URI
    artifact_uri = mlflow.get_artifact_uri()
    print(f"Artifacts stored at: {artifact_uri}")
    
    # Get URI for specific artifact
    model_uri = mlflow.get_artifact_uri("model.pkl")

# Download artifacts from a run
from mlflow.artifacts import download_artifacts

local_path = download_artifacts(
    run_id="abc123",
    artifact_path="model"
)
```

## Best Practices

<AccordionGroup>
  <Accordion title="Use Context Managers">
    Always use `with mlflow.start_run()` instead of manually calling `start_run()` and `end_run()`. This ensures runs are properly closed even if exceptions occur.
  </Accordion>
  
  <Accordion title="Set Experiment Names">
    Use `mlflow.set_experiment()` to organize related runs. On platforms like Databricks, use absolute paths: `/Users/<username>/my-experiment`.
  </Accordion>
  
  <Accordion title="Log Early and Often">
    Log parameters at the start of your run, and log metrics throughout training. This makes debugging easier if runs fail.
  </Accordion>
  
  <Accordion title="Use Descriptive Run Names">
    Set meaningful run names using the `run_name` parameter in `start_run()` to make runs easier to identify.
  </Accordion>
</AccordionGroup>
