## What is an MLflow Model?

An MLflow Model is a standard format for packaging machine learning models that can be used in a variety of downstream tools. The format defines a convention for saving models in different "flavors" that can be understood by different deployment tools.

Each MLflow Model is a directory containing:
- An `MLmodel` file that defines the model's flavors and metadata
- Model artifacts (the actual model files)
- Dependency specifications (conda.yaml, requirements.txt, python_env.yaml)
- Optional input examples and signatures

## Model Flavors

MLflow Models support multiple "flavors" - different ways the model can be interpreted and used. Built-in flavors include:

- `python_function` (pyfunc) - Generic Python function interface
- `sklearn` - Scikit-learn models
- `pytorch` - PyTorch models
- `tensorflow` - TensorFlow models
- `keras` - Keras models
- `xgboost` - XGBoost models
- `lightgbm` - LightGBM models
- `transformers` - Hugging Face Transformers
- `langchain` - LangChain chains and agents
- `openai` - OpenAI models
- And many more...

## Logging Models

### Basic Model Logging

Each flavor provides a `log_model()` function:

```python
import mlflow
import mlflow.sklearn
from sklearn.ensemble import RandomForestClassifier

# Train a model
model = RandomForestClassifier(n_estimators=100)
model.fit(X_train, y_train)

# Log the model
with mlflow.start_run():
    mlflow.sklearn.log_model(
        sk_model=model,
        artifact_path="model",
        input_example=X_train[:5],
        registered_model_name="random-forest-classifier"
    )
```

### Model Signature

Signatures define the expected input and output schema:

```python
import mlflow
from mlflow.models import infer_signature
import pandas as pd

# Infer signature from data
predictions = model.predict(X_train)
signature = infer_signature(X_train, predictions)

with mlflow.start_run():
    mlflow.sklearn.log_model(
        sk_model=model,
        artifact_path="model",
        signature=signature
    )
```

<Warning>
Models logged to Databricks Unity Catalog require signatures. Set `input_example` to auto-infer signatures.
</Warning>

## PyFunc - Universal Model Interface

The `pyfunc` flavor provides a universal interface that all MLflow models implement:

```python
import mlflow.pyfunc

# Load any model as pyfunc
model = mlflow.pyfunc.load_model("runs:/abc123/model")

# Predict with various input types
import pandas as pd
import numpy as np

# Pandas DataFrame
df_input = pd.DataFrame({"x1": [1, 2], "x2": [3, 4]})
predictions = model.predict(df_input)

# NumPy array
np_input = np.array([[1, 3], [2, 4]])
predictions = model.predict(np_input)

# Python dict
dict_input = {"x1": [1, 2], "x2": [3, 4]}
predictions = model.predict(dict_input)
```

## Custom PyFunc Models

Create custom models by implementing the `PythonModel` interface:

```python
import mlflow
from mlflow.pyfunc import PythonModel

class CustomModel(PythonModel):
    def load_context(self, context):
        """Load model artifacts."""
        import joblib
        self.model = joblib.load(context.artifacts["model_path"])
    
    def predict(self, context, model_input):
        """Generate predictions."""
        return self.model.predict(model_input)

# Log custom model
with mlflow.start_run():
    mlflow.pyfunc.log_model(
        artifact_path="custom_model",
        python_model=CustomModel(),
        artifacts={"model_path": "path/to/model.pkl"},
        conda_env="conda.yaml"
    )
```

## Model Metadata - ModelInfo

When logging a model, MLflow returns a `ModelInfo` object with metadata:

```python
import mlflow

with mlflow.start_run() as run:
    model_info = mlflow.sklearn.log_model(
        sk_model=model,
        artifact_path="model"
    )
    
    print(f"Model URI: {model_info.model_uri}")
    print(f"Run ID: {model_info.run_id}")
    print(f"Artifact path: {model_info.artifact_path}")
    print(f"Flavors: {model_info.flavors}")
    print(f"Model UUID: {model_info.model_uuid}")
    print(f"Signature: {model_info.signature}")
```

## Model Configuration

Pass configuration to models at inference time:

```python
import mlflow
from mlflow.models import ModelConfig

# Define configuration
config = ModelConfig(
    development_config={
        "temperature": 0.7,
        "max_tokens": 100
    }
)

# Use in custom models
class ConfigurableModel(mlflow.pyfunc.PythonModel):
    def predict(self, context, model_input, params=None):
        temperature = params.get("temperature", 0.5) if params else 0.5
        return self.model.generate(model_input, temperature=temperature)
```

## Logged Models API

MLflow 2.x introduces the Logged Models API for managing models during training:

### Initialize Logged Model

```python
import mlflow

with mlflow.start_run():
    # Initialize a model that will be logged incrementally
    logged_model = mlflow.initialize_logged_model(
        artifact_path="model",
        model_type="pytorch"
    )
    
    # Train and log checkpoints
    for epoch in range(100):
        train_epoch()
        
        # Update model periodically
        if epoch % 10 == 0:
            save_checkpoint(f"checkpoint-{epoch}.pt")
            mlflow.log_artifact(f"checkpoint-{epoch}.pt", "checkpoints")
    
    # Finalize the model
    mlflow.pytorch.log_model(
        pytorch_model=final_model,
        artifact_path="model"
    )
    mlflow.finalize_logged_model(logged_model)
```

### Get Logged Model Info

```python
import mlflow

with mlflow.start_run():
    model_info = mlflow.sklearn.log_model(model, "model")
    
    # Later in the same run
    logged_model = mlflow.get_logged_model(model_info.model_id)
    print(f"Model status: {logged_model.status}")
```

## Model Dependencies

### Automatic Dependency Inference

MLflow automatically infers and logs dependencies:

```python
import mlflow

# Dependencies are automatically captured
with mlflow.start_run():
    mlflow.sklearn.log_model(
        sk_model=model,
        artifact_path="model"
        # Creates conda.yaml, requirements.txt, python_env.yaml automatically
    )
```

### Custom Dependencies

```python
import mlflow

# Specify custom conda environment
conda_env = {
    "channels": ["defaults", "conda-forge"],
    "dependencies": [
        f"python={python_version}",
        "pip",
        {
            "pip": [
                "mlflow",
                "scikit-learn==1.3.0",
                "pandas==2.0.0"
            ]
        }
    ]
}

with mlflow.start_run():
    mlflow.sklearn.log_model(
        sk_model=model,
        artifact_path="model",
        conda_env=conda_env
    )
```

## Model Registry Integration

### Register Model During Logging

```python
import mlflow

with mlflow.start_run():
    mlflow.sklearn.log_model(
        sk_model=model,
        artifact_path="model",
        registered_model_name="my-sklearn-model"
    )
```

### Register After Logging

```python
import mlflow

# Log model
with mlflow.start_run() as run:
    mlflow.sklearn.log_model(model, "model")

# Register separately
model_uri = f"runs:/{run.info.run_id}/model"
mlflow.register_model(model_uri, "my-sklearn-model")
```

## Best Practices

<AccordionGroup>
  <Accordion title="Always Include Signatures">
    Model signatures validate inputs and document expected schemas. Use `input_example` for automatic inference:
    
    ```python
    mlflow.sklearn.log_model(
        sk_model=model,
        artifact_path="model",
        input_example=X_train[:5]  # Auto-infers signature
    )
    ```
  </Accordion>
  
  <Accordion title="Use Input Examples">
    Input examples serve as documentation and test cases:
    
    ```python
    input_example = pd.DataFrame({
        "age": [25, 30, 35],
        "income": [50000, 60000, 70000]
    })
    
    mlflow.sklearn.log_model(
        sk_model=model,
        artifact_path="model",
        input_example=input_example
    )
    ```
  </Accordion>
  
  <Accordion title="Version Your Dependencies">
    Pin dependency versions for reproducibility:
    
    ```python
    pip_requirements = [
        "scikit-learn==1.3.0",
        "pandas==2.0.0",
        "numpy==1.24.0"
    ]
    
    mlflow.sklearn.log_model(
        sk_model=model,
        artifact_path="model",
        pip_requirements=pip_requirements
    )
    ```
  </Accordion>
  
  <Accordion title="Use Meaningful Artifact Paths">
    Organize models with clear artifact paths:
    
    ```python
    # Good: descriptive paths
    mlflow.sklearn.log_model(model, "models/random_forest")
    mlflow.pytorch.log_model(model, "models/resnet50")
    
    # Avoid: generic paths
    mlflow.sklearn.log_model(model, "model")
    ```
  </Accordion>
</AccordionGroup>
